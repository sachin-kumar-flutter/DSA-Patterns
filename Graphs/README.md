# Graph Algorithms

This section contains problems based on graph traversal techniques such as BFS and DFS.

## 🧠 What is a Graph?

A graph is a collection of nodes (vertices) connected by edges.  
Many problems can be modeled as graphs—even if they are given as grids or matrices.

---

## 🔑 Key Patterns Covered

- Breadth-First Search (BFS)
- Depth-First Search (DFS)
- Multi-Source BFS
- Shortest Path (unweighted graphs)

---

## 📂 Problems Included

### 1. Rotting Oranges
- Pattern: Multi-Source BFS
- Idea: Treat all rotten oranges as sources and spread infection level by level
- Key Concept: BFS layers represent time (minutes)

---

## 🎯 Why Graphs?

- Helps model real-world problems like networks, grids, and dependencies  
- BFS is used for shortest path in unweighted graphs  
- DFS helps in traversal, cycles, and connectivity  

---

## ⚡ Key Takeaway

Use BFS when:
- You need the shortest time or minimum steps  
- Multiple sources spread simultaneously  

Use DFS when:
- You explore all possible paths  
- You need recursion or backtracking
