# 🔗 Hash Table Patterns

This section contains problems solved using **Hashing (unordered_map / unordered_set)** — one of the most powerful tools for optimizing time complexity.

---

## 🚀 Why Hash Tables?

Hash tables allow:
- ⚡ O(1) average lookup time
- Efficient frequency counting
- Eliminating nested loops
- Solving problems in linear time

---

## 🧠 Key Patterns Covered

- Frequency Counting
- Presence Checking
- First Occurrence Tracking
- Longest/Shortest Sequence Detection
- Duplicate Detection & Removal

---

## 📂 Problems Included

### 1. First Element with Unique Frequency
- Use hashmap to count frequencies
- Track the first element with unique frequency

### 2. Longest Consecutive Sequence
- Convert array into unordered_set
- Start sequence only when `(num - 1)` is not present
- Expand forward to count streak
- Achieves **O(n)** time complexity

---

## 💡 Key Insight

> "Hashing reduces time complexity by trading space for speed."
