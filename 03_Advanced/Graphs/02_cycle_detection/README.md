# 🔄 Cycle Detection in Graphs

> Detect cycles, identify safe paths, and understand the backbone of many graph algorithms.

This directory focuses on one of the most important graph concepts—**Cycle Detection**. Whether it's checking if a graph contains a cycle, determining if tasks can be completed, or identifying nodes that are eventually safe, these problems strengthen your understanding of graph traversal and dependency analysis.

---

## 📚 Problems Covered

- 📖 Course Schedule
- 🛡️ Find Eventual Safe States

---

## 🎯 Concepts Practiced

- Cycle Detection in Directed Graphs
- DFS with Recursion Stack
- Node Coloring Technique
- Topological Sort (Kahn's Algorithm)
- Reverse Graph Traversal
- In-Degree Computation
- Dependency Resolution

---

## 💡 What You'll Learn

- How to detect cycles efficiently
- Difference between visited array and recursion stack
- Why DAGs (Directed Acyclic Graphs) are special
- Finding nodes that are part of or lead to cycles
- Solving dependency-based problems using graph algorithms
- Time and Space Complexity analysis

## 🚀 Why These Problems?

Many real-world systems—from **course prerequisites** and **build systems** to **package managers** and **task scheduling**—depend on cycle detection.

Master this pattern, and you'll unlock a whole category of interview problems involving dependencies and directed graphs.

> **"Every impossible schedule hides a cycle. Find it, and the graph reveals the truth."** 🔄
