# 📊 Arrays - DSA Patterns

## 🔑 Key Patterns Covered

✨ **In-place Modification**
→ Modify the array itself to store useful information without extra space

✨ **Prefix & Suffix Computation**
→ Precompute values to avoid repeated calculations

✨ **Two Pointers**
→ Use multiple indices to efficiently traverse and reduce complexity

✨ **Monotonic Segments (Segment-Based)**
→ Break array into continuous segments where a condition holds

✨ **Space Optimization Techniques**
→ Reduce memory usage while maintaining efficiency

---

## 📂 Subcategories

### 🔹 In-place Marking

* Encode information directly within the array
* Avoid using extra data structures
* Common in:

  * Missing numbers
  * Duplicate detection

---

### 🔹 Prefix-Suffix Technique

* Store cumulative values (prefix/suffix arrays)
* Helps in:

  * Range queries
  * Product/sum problems
  * Optimization of repeated operations

---

### 🔹 Two Pointers

* Use two indices to traverse array efficiently
* Reduces nested loops to linear scans
* Common in:

  * Sorted arrays
  * Pair/triplet problems
  * Sliding window variations

---

### 🔹 Monotonic Segments (Segment-Based)

* Divide array into segments based on a condition

* Process each segment mathematically

* Useful for:

  * Increasing/decreasing subarrays
  * Consecutive elements problems
  * Counting subarrays efficiently

---

## 🎯 Why Arrays?

💡 Arrays are the **foundation of DSA**

* 📌 Most commonly asked in interviews

## ⚡ Key Takeaways

✔️ Minimize extra space whenever possible
✔️ Avoid recomputation using preprocessed values
✔️ Use pointers to reduce time complexity
✔️ Think in terms of patterns, not individual problems

---

## 🚀 Mental Model

When you see an array problem, ask yourself:

> 🔍 Can I reuse this space?
> 🔍 Can I precompute something?
> 🔍 Can I use two pointers?
> 🔍 Can I break this into segments?
> 🔍 Am I repeating work unnecessarily?
