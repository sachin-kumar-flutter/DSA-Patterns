# 🧠 Problem Solving Approach

A structured way to approach any DSA problem in interviews.

---

## 🚀 Step 1: Understand the Problem

* Read carefully
* Identify input/output
* Clarify constraints
* Ask questions if needed

👉 Never rush this step.

---

## 🧩 Step 2: Think of Brute Force

* Start with a simple approach
* Don’t worry about optimization initially

👉 Shows clarity and logical thinking.

---

## ⚡ Step 3: Optimize the Solution

Ask:

* Can I reduce time complexity?
* Can I use a known pattern?

👉 Common optimizations:

* Hashing
* Sliding Window
* Binary Search
* Prefix Sum

---

## 🧠 Step 4: Identify the Pattern

* Subarray → Sliding Window
* Sorted → Two Pointers / Binary Search
* Frequency → Hash Map
* All combinations → Backtracking

👉 Pattern recognition is key.

---

## ✍️ Step 5: Write Clean Code

* Use meaningful variable names
* Keep code simple and readable
* Avoid unnecessary complexity

---

## ⏱️ Step 6: Analyze Complexity

* Time Complexity
* Space Complexity

👉 Always explain your reasoning.

---

## 🔍 Step 7: Dry Run

* Test with sample inputs
* Check edge cases

Examples:

* Empty input
* Single element
* Large values

---

## 🗣️ Step 8: Communicate Clearly

* Explain your approach
* Think out loud
* Be confident

👉 Communication matters as much as coding.

---

## ⚠️ Common Pitfalls

* Skipping brute force
* Jumping to coding
* Not testing edge cases

---

## 💡 Key Insight

👉 The goal is not just to solve the problem,
but to show **how you think**.

---

## 🧾 Final Strategy

1. Understand
2. Brute force
3. Optimize
4. Code
5. Test

---

Solve smart. Think clearly. Communicate well. 🚀
