# Greedy Algorithms

This section contains carefully selected problems that demonstrate the power of greedy strategies.

## 🧠 What is Greedy?

Greedy algorithms make locally optimal choices at each step with the hope of achieving a globally optimal solution.

---

## 🔑 Key Patterns Covered

- Two Pointers + Greedy Matching
- Sorting + Optimal Pairing
- Reverse Processing (Thinking Backwards)

---

## 📂 Problems Included

### 1. Minimize Maximum Pair Sum
- Pattern: Greedy + Two Pointers
- Idea: Pair smallest with largest to balance sums

### 2. Police and Thieves
- Pattern: Greedy + Two Pointers
- Idea: Match nearest valid policeman–thief pairs

### 3. Sum of Matrix After Queries
- Pattern: Greedy + Reverse Processing
- Idea: Process queries from end since only last updates matter

---

## 🎯 Why These Problems?

These problems are selected to highlight:
- Core greedy intuition
- Optimization strategies
- Real interview patterns

---

## ⚡ Key Takeaway

Greedy works when:
- A local optimal choice leads to global optimal solution
- The problem has a clear structure that avoids reconsideration of past decisions
