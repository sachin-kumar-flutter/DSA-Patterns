# 📚 Stack

## 📌 What is a Stack?

A stack is a linear data structure that follows the **LIFO (Last In, First Out)** principle.

👉 Think of a stack like a pile of plates — the last plate placed is the first one to be removed.

---

## ⚡ Core Operations

* **push(x)** → Insert element at the top
* **pop()** → Remove top element
* **top() / peek()** → Access top element
* **isEmpty()** → Check if stack is empty

---

## ⏱️ Time Complexities

| Operation | Complexity |
| --------- | ---------- |
| Push      | O(1)       |
| Pop       | O(1)       |
| Top       | O(1)       |

---

## 🧠 Implementation Options

* Using **Arrays**
* Using **Linked List**
* Using **STL (`stack<int>`) in C++**

---

## 🎯 When to Use Stack?

* Reversing data
* Undo/Redo operations
* Expression evaluation (infix, postfix)
* Backtracking
* Function call management (recursion)

---

## 🔥 Important Problems

* Valid Parentheses
* Next Greater Element
* Largest Rectangle in Histogram
* Min Stack
* Evaluate Postfix Expression

---

## ⚠️ Common Pitfalls

* Stack Overflow (when full in array implementation)
* Stack Underflow (pop on empty stack)

---

## 💡 Key Insight

Stacks are heavily used in **pattern-based problems**, especially:

* Monotonic Stack
* Expression Parsing
* DFS (implicit via recursion)

---

## 🧾 Notes

Stack is one of the simplest yet most powerful data structures.
Mastering it unlocks many medium-to-hard level interview problems.
