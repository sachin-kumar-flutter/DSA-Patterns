# 🌳 Trees

## 📌 What is a Tree?

A tree is a hierarchical data structure consisting of **nodes connected by edges**.

👉 Think of it like a family tree — one root, multiple branches, and leaf nodes at the ends.

---

## 🧠 Basic Terminology

* **Root** → Topmost node
* **Parent / Child** → Relationship between nodes
* **Leaf Node** → Node with no children
* **Height** → Longest path from root to leaf
* **Depth** → Distance from root

---

## 🌿 Types of Trees

* Binary Tree
* Binary Search Tree (BST)
* Balanced Trees (AVL, Red-Black)
* Heap (Special type of tree)
* Trie

---

## ⚡ Binary Tree Properties

* Each node has **at most 2 children**
* Left child and right child

---

## 🔍 Tree Traversals

### Depth First Traversal (DFS)

* **Inorder (LNR)**
* **Preorder (NLR)**
* **Postorder (LRN)**

### Breadth First Traversal (BFS)

* Level Order Traversal

---

## ⏱️ Time Complexities

| Operation    | Complexity |
| ------------ | ---------- |
| Traversal    | O(n)       |
| Search (BT)  | O(n)       |
| Search (BST) | O(log n)*  |

* If balanced

---

## 🎯 When to Use Trees?

* Hierarchical data representation
* Searching and sorting (BST)
* Expression parsing
* File systems
* Databases

---

## 🔥 Important Problems

* Height of Tree
* Diameter of Tree
* Lowest Common Ancestor (LCA)
* Level Order Traversal
* Validate BST
* Maximum Path Sum

---

## ⚠️ Common Pitfalls

* Confusing height vs depth
* Not handling NULL nodes properly
* Forgetting base cases in recursion

---

## 💡 Key Insight

Trees are all about **recursion and structure**.

👉 If you understand recursion well, trees become much easier.
