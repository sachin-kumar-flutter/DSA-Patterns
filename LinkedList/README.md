# 🔗 Linked List Patterns

This section covers problems based on **Linked Lists**, focusing on pointer manipulation and in-place operations.

---

## 🚀 Why Linked Lists?

Linked Lists are fundamental for:
- Efficient insertions/deletions
- Understanding memory & pointers
- Solving in-place transformation problems

---

## 🧠 Key Patterns Covered

- Two Pointer Technique (Slow & Fast)
- In-place Reversal
- Dummy Node Usage
- Pointer Rewiring
- Cycle Detection
- Merging Lists

---

## 📂 Problems Included

### 1. Reverse Linked List
- Use three pointers: `prev`, `curr`, `next`
- Reverse links iteratively
- Classic pointer manipulation problem

---

### 2. Add Two Numbers
- Simulate addition with carry
- Traverse both lists simultaneously
- Use dummy node for easier handling

---

### 3. Odd Even Linked List
- Separate nodes based on position (odd/even)
- Maintain two pointers and reconnect
- Done in-place with O(1) space

---

## 💡 Key Insights

> "Linked List problems are about pointer control, not data manipulation."

- Always draw the list before coding ✍️  
- Think in terms of links, not values  
- Use dummy nodes to simplify edge cases  

---

## 🛠️ Complexity Cheatsheet

| Operation              | Time Complexity |
|-----------------------|----------------|
| Traversal             | O(n)           |
| Insertion (given node)| O(1)           |
| Deletion (given node) | O(1)           |
